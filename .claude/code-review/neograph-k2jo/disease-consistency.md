## Notes

**Dominant (canonical) form**: `src/neograph/_oracle.py` is the single source of truth — `make_oracle_redirect_fn`, `make_eachoracle_redirect_fn`, `make_each_redirect_fn` for redirect, and `make_oracle_merge_fn` for the merge barrier. Every other Oracle wiring site routes through these: `compiler.py:429/432` (sub-construct Oracle), `compiler.py:526/529` (node-level Oracle), and `_wiring.py:203/262` (Each×Oracle redirect + the *outer* group barrier all call the canonical factories). `factory.py:43-45` only re-exports. So the canonical form is overwhelmingly dominant — the disease is a single localized cluster.

**The one minority form** is the body of `_merge_one_group` (`_wiring.py:303-372`), which the Each×Oracle group barrier (`group_merge_barrier`, line 268-294) calls per-key instead of delegating to the canonical merge core. All three table rows are facets of this one function: the duplicate helper itself (#1) and its two re-derived dispatch branches (#2 LLM-judge, #3 scripted). They are counted separately because the spec's disposition table wants reproducible line anchors, but they MIGRATE together as one consolidation.

**Why not ALLOWLIST**: the fork is not justified by an irreducible structural difference. The real difference is only the input shape — canonical reads `collector_field` from `state` and unwraps via `_unwrap_oracle_results`; the fused path already holds a per-group `variants` list. That is a clean seam: extract a `variants -> merged` core (taking `oracle, variants, output_model, config, runtime, scripted_lookup` and an optional state-for-DI) and have both `make_oracle_merge_fn`'s inner closures and `_merge_one_group` call it. The documented `None`-state DI caveat (#3) becomes a parameter, not a justification for two copies.

**Consolidation target**: ARCH-1 / neograph-s0iz. Recommend the shared core live in `_oracle.py` (the canonical home), keeping `_wiring.py` as a pure topology layer per the module's own docstring ("lower modifiers into LangGraph topology"). Note the layer rule in AGENTS.md: teach the single helper, do not re-inline the algorithm — the same discipline that produced `effective_producer_type`.
