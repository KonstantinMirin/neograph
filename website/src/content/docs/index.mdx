**neograph is the fastest way to build production-grade agents on [LangGraph](https://github.com/langchain-ai/langgraph).** You declare your agent as typed Python functions; neograph infers the graph, validates every input and output at assembly time, and compiles to a LangGraph state machine with durable checkpointing, structured observability, and tool orchestration built in.

It's for real-world execution — not vibes. Every edge in your graph is a typed Pydantic contract. Every run is checkpointed and resumable. Every node emits structured logs. You focus on the agent logic; the plumbing is handled.

- **Typed end to end.** Inputs and outputs are Pydantic models. Parameter names are dependencies. Type mismatches fail at assembly time, not in production at 2 a.m.
- **Durable by default.** Compiles to standard LangGraph — you get checkpointing, interrupts, streaming, and resume-from-failure for free.
- **Observable out of the box.** Structured logs on every node. Trace providers and shared resources flow through `Annotated[T, FromConfig]` without polluting signatures.
- **No wiring.** A function is a node. A parameter name is an edge. An `if` is a branch. You write the logic; the graph assembles itself.
- **Three APIs, one compiler.** `@node` for most agents. `ForwardConstruct` for Python `if`/`for`/`try` control flow. Programmatic `|` pipe for runtime construction.
- **No lock-in.** `mode='raw'` is a first-class escape hatch to raw LangGraph.

## Your first agent

```python
import sys
from pydantic import BaseModel
from neograph import node, construct_from_module, compile, run

# Types — plain Pydantic.
class Claims(BaseModel):
    items: list[str]

class Classified(BaseModel):
    security: list[str]
    performance: list[str]

class Report(BaseModel):
    summary: str

# Agent — plain Python functions. Types flow through the signatures.
@node(output=Claims, prompt='decompose', model='reason')
def decompose(topic: str) -> Claims: ...

@node(output=Classified, prompt='classify', model='fast')
def classify(decompose: Claims) -> Classified: ...

@node(output=Report, prompt='summarize', model='fast')
def summarize(classify: Classified) -> Report: ...

# Compile and run.
pipeline = construct_from_module(sys.modules[__name__])
graph = compile(pipeline)

result = run(graph, input={'topic': 'microservice authentication'})
print(result['summarize'].summary)
```

That's the whole agent. Three functions, three types, three lines of compilation. No `TypedDict`, no `add_node`, no `add_edge`, no state-dict reads and writes.

The parameter name `decompose` in `classify(decompose: Claims)` IS the dependency. neograph reads the signature, builds the DAG, validates that `decompose` produces `Claims`, and wires `decompose → classify → summarize` automatically. Rename the `decompose` function and the downstream parameter breaks at import time — your IDE will tell you before you even hit save.

Fan-in is just more parameters:

```python
@node(output=Report, prompt='report', model='reason')
def report(claims: Claims, scores: Scores, verified: VerifyResult) -> Report: ...
```

## Control flow is Python

For agents with real branching logic — retries, quality gates, conditional paths — use `ForwardConstruct`:

```python
from neograph import ForwardConstruct, Node, compile

class Analysis(ForwardConstruct):
    check   = Node(output=CheckResult, prompt='check', model='fast')
    deep    = Node(output=Result, prompt='deep', model='reason')
    shallow = Node(output=Result, prompt='quick', model='fast')

    def forward(self, topic):
        checked = self.check(topic)
        if checked.confidence > 0.8:
            return self.shallow(checked)
        return self.deep(checked)

graph = compile(Analysis())
```

The `if` compiles to a LangGraph conditional edge. `for` compiles to fan-out. Python is the graph language — your type checker sees every branch, your debugger works, and you can call `forward()` directly in tests with fake LLMs.
