**NeoGraph is a declarative LLM graph compiler for Python.** You define your pipeline as typed Python functions. NeoGraph reads your function signatures, infers the DAG from parameter names, validates types at assembly time, and compiles to [LangGraph](https://github.com/langchain-ai/langgraph) — with checkpointing, observability, and tool orchestration.

The key features are:

- 🐍 **Zero wiring.** A function is a node. A parameter name is an edge. An `if` is a branch. No `add_node`, no `add_edge`, no router functions.
- 🛡️ **Type-safe.** Assembly-time validation catches mismatches before you run. Your IDE sees every dependency. Rename a function — downstream breaks at import time.
- ⚡ **Production-ready.** Compiles to standard LangGraph. Full access to checkpointing, streaming, interrupts, and the LangGraph ecosystem.
- 🔀 **Three APIs, one compiler.** `@node` for most pipelines. `ForwardConstruct` for Python `if`/`for`/`try` control flow. Programmatic `|` pipe for runtime construction.
- 🧪 **Testable at every level.** `run_isolated()` for unit tests. Direct `forward()` call for debugging. Fake LLMs without mocking.
- 🪝 **No lock-in.** Drop down to raw LangGraph whenever you need to. `mode='raw'` is a first-class escape hatch.

## Installation

```bash
# uv (recommended)
uv add neograph

# pip
pip install neograph
```

## Example

```python
import sys
from neograph import node, construct_from_module, compile, run

@node(output=Claims, prompt='decompose', model='reason')
def decompose(topic: RawText) -> Claims: ...

@node(output=Classified, prompt='classify', model='fast')
def classify(decompose: Claims) -> Classified: ...

@node(output=Summary, prompt='summarize', model='fast')
def summarize(classify: Classified) -> Summary: ...

pipeline = construct_from_module(sys.modules[__name__])
graph = compile(pipeline)
result = run(graph, input={'topic': 'microservice authentication'})
```

The parameter name `decompose` in `classify(decompose: Claims)` IS the dependency. No `nodes=[...]` list, no ordering, no wiring. Fan-in is just more parameters: `def report(claims, scores, verified)`.

## Control flow is Python

```python
from neograph import ForwardConstruct, Node, compile

class Analysis(ForwardConstruct):
    check   = Node(output=CheckResult, prompt='check', model='fast')
    deep    = Node(output=Result, prompt='deep', model='reason')
    shallow = Node(output=Result, prompt='quick', model='fast')

    def forward(self, topic):
        checked = self.check(topic)
        if checked.confidence > 0.8:
            return self.shallow(checked)
        return self.deep(checked)

graph = compile(Analysis())
```

The `if` compiles to a LangGraph conditional edge. `for` compiles to Each fan-out. Python is the graph language — your type checker sees every branch, your debugger works, and you can call `forward()` directly in tests with fakes.

## What you don't write

| You skip | Because NeoGraph |
|----------|-----------------|
| `TypedDict` state schemas | Infers state from your node output types |
| `add_node` / `add_edge` calls | Reads dependencies from parameter names |
| State dict reads and writes | Passes typed values directly to your functions |
| Router functions for branches | Compiles Python `if` to conditional edges |
| `Send` scaffolding for fan-out | Compiles `map_over=...` to the same IR |
| Ensemble boilerplate | Compiles `ensemble_n=3, merge_fn=...` to fan-out + barrier + merge |
| Condition registries | Accepts inline lambdas via `interrupt_when=` |
