NeoGraph exposes three API surfaces that compile to the same IR:

1. **`@node` API** — the primary surface for human-written pipelines
2. **`ForwardConstruct`** — class-based with Python control flow
3. **Programmatic / IR** — `Node` + `Construct` + `|` pipe for runtime construction

All three are importable from `neograph`:

```python
from neograph import (
    # Primary @node API
    node, construct_from_module, construct_from_functions,
    FromInput, FromConfig, merge_fn,
    # ForwardConstruct
    ForwardConstruct,
    # Programmatic IR
    Node, Construct, Oracle, Each, Operator,
    # Tools
    Tool, tool, ToolInteraction,
    # BAML rendering
    describe_type, describe_value,
    # Prompt inspection + standalone compilation
    render_prompt, compile_prompt,
    # Renderers (opt-in alternatives to BAML)
    Renderer, XmlRenderer, DelimitedRenderer, JsonRenderer,
    # Shared
    compile, run,
    RenderedInput, build_rendered_input, ExcludeFromOutput,
    # Errors
    NeographError, ConstructError, CompileError, ConfigurationError,
    CheckpointSchemaError, ExecutionError,
    # Spec loader
    load_spec, register_type, lookup_type,
)
```

## @node API

### `@node(...)`

Decorator that turns a function into a Node with signature-inferred dependencies.

```python
@node(
    outputs: type[BaseModel] | dict[str, type],
    *,
    mode: Literal["scripted", "think", "agent", "act", "raw"] | None = None,
    prompt: str | None = None,
    model: str | None = None,
    tools: list[Tool] | None = None,
    llm_config: dict | None = None,
    name: str | None = None,
    # Modifier kwargs
    map_over: str | None = None,
    map_key: str | None = None,
    ensemble_n: int | None = None,
    models: list[str] | None = None,
    merge_fn: str | None = None,
    merge_prompt: str | None = None,
    merge_pre_process: Callable | None = None,
    merge_post_process: Callable | None = None,
    merge_fallback: Callable | None = None,
    interrupt_when: str | Callable | None = None,
    gate_tools_when: str | Callable | None = None,
    loop_when: str | Callable | None = None,
    max_iterations: int | None = None,
    on_exhaust: "error" | "last" | None = None,
)
```

- **`outputs`** — Pydantic model or `dict[str, type]` the node produces. Required for all modes except `raw`. Dict form enables multi-output (e.g. `{"result": Claims, "tool_log": list[ToolInteraction]}`).
- **`mode`** — Execution mode. Inferred from kwargs if not set: `prompt=` + `model=` → `think`; neither → `scripted`.
- **`prompt`**, **`model`** — Required for LLM modes (`think`, `agent`, `act`). Validated at decoration time.
- **`tools`** — List of `Tool` instances for `agent` / `act` modes.
- **`llm_config`** — Per-node LLM settings (temperature, max_tokens, output_strategy, etc.)
- **`name`** — Override the node name. Default: function name with `_` → `-`.
- **`inputs`** — Explicit `dict[str, type]` input spec. Usually inferred from param annotations.
- **`map_over`** / **`map_key`** — Fan-out: run this node once per item in the referenced collection.
- **`ensemble_n`** / **`merge_fn`** / **`merge_prompt`** — Oracle ensemble: N parallel generators plus a scripted or LLM merge.
- **`merge_pre_process`** — `fn(variants) -> dict`. Replaces the default input_data for the merge prompt. Only valid with `merge_prompt`.
- **`merge_post_process`** — `fn(result, variants) -> result`. Transforms the LLM merge result before state write. Skipped when fallback fires.
- **`merge_fallback`** — `fn(variants, error) -> result`. Catches LLM errors during merge and returns a deterministic result.
- **`models`** — Multi-model ensemble: list of model tiers. Each generator gets a different model round-robin. Infers `ensemble_n` from `len(models)`. When `models=` is set without `merge_fn`/`merge_prompt`, the function body IS the merge.
- **`interrupt_when`** — Human-in-the-loop: inline callable or registered condition name.
- **`gate_tools_when`** — Tool-approval gate for `agent` / `act` nodes only. A callable (or registered condition name) receiving the full state; it pauses BEFORE the synthesized `{node}__tools` superstep so a human approves before any tool side effect runs. A truthy return is the interrupt payload shown to the human; a falsy return lets the tools run without pausing. On resume, only an explicit `{"approved": True}` proceeds — any other payload is fail-closed as a denial, which appends a denial `ToolMessage` and routes back to the agent to keep reasoning. Setting it on a non-agent/act node raises at construction.
- **`loop_when`** — Loop: callable or registered condition name. Return `True` to continue looping. The value may be `None` on the first iteration, so the callable must be None-safe.
- **`max_iterations`** — Loop iteration cap. Default 10. Used with `loop_when=`.
- **`on_exhaust`** — `"error"` (default) raises `ExecutionError` when max reached; `"last"` exits with last result.
- **`renderer`** — Pluggable prompt-input renderer (`XmlRenderer`, `DelimitedRenderer`, `JsonRenderer`, or custom).
- **`context`** — `list[str]` of state field names injected verbatim into the prompt (not BAML-rendered). For pre-formatted context like graph catalogs.
- **`skip_when`** — Callable predicate. When True, skips the LLM call entirely.
- **`skip_value`** — Callable that produces the output when `skip_when` fires.

The function body runs for `scripted` and `raw` modes. For LLM modes, the body is dead code — use `...` as the body.

### `construct_from_module(mod, name=None, *, llm_config=None, input=None, output=None)`

Walks a module's `@node`-decorated functions, infers dependencies from parameter names, topologically sorts, and returns a `Construct`.

```python
pipeline = construct_from_module(sys.modules[__name__], name="my-pipeline")
```

When building a sub-construct, pass `input=` / `output=` to define the state boundary:

```python
sub = construct_from_module(mod, input=VerifyClaim, output=ClaimResult)
```

### `construct_from_functions(name, functions, *, llm_config=None, input=None, output=None)`

Builds a `Construct` from an explicit list of `@node`-decorated functions and/or `Construct` instances. Use when multiple pipelines share a file or when mixing `@node` functions with sub-constructs:

```python
pipeline = construct_from_functions("rw-ingestion", [
    flatten_claims,
    verify_claim.map("flatten_claims.claims", key="claim_id"),
    deterministic_merge,
])
```

Accepts both `@node` items and `Construct` items in the same list. Construct items must declare `output=` so downstream `@node` params can resolve against them.

### `@merge_fn`

Decorator for Oracle merge functions with `FromInput`/`FromConfig` DI support:

```python
@merge_fn
def combine(variants: list[Claims], shared: Annotated[Ctx, FromConfig]) -> Claims:
    return Claims(items=[i for v in variants for i in v.items])

@node(outputs=Claims, ensemble_n=3, merge_fn="combine")
def generate() -> Claims: ...
```

The first parameter receives the list of Oracle generator results. Subsequent parameters use standard DI annotations. The merge_fn's first parameter type (`list[T]`) tells the framework what type each generator should produce — the generators use `T` as the LLM schema, not `node.outputs`.

### `FromInput`

Dependency-injection marker used inside `typing.Annotated`. Parameters annotated with `Annotated[T, FromInput]` are resolved from `run(input={...})` at runtime (runtime-variable data — user queries, document IDs, per-call configuration).

```python
from typing import Annotated
from neograph import node, FromInput

@node(outputs=Report)
def summarize(claims: Claims, topic: Annotated[str, FromInput]) -> Report: ...

run(graph, input={'topic': 'security', 'node_id': 'demo'})
```

If the inner type `T` is a Pydantic `BaseModel` subclass, `FromInput` bundles: it constructs an instance by pulling each of the model's declared fields from `config['configurable']`. Use this to inject a typed context object into many nodes without repeating per-field parameters.

### `FromConfig`

Dependency-injection marker used inside `typing.Annotated`. Parameters annotated with `Annotated[T, FromConfig]` are resolved from `config['configurable']` at runtime (shared infrastructure — rate limiters, database connections, trace providers).

```python
from typing import Annotated
from neograph import node, FromConfig

@node(outputs=Result)
def process(data: Data, tracer: Annotated[Tracer, FromConfig]) -> Result: ...

run(graph, input={...}, config={'configurable': {'tracer': my_tracer}})
```

`FromConfig` also bundles for Pydantic models — `Annotated[Shared, FromConfig]` constructs a `Shared` instance from per-field `configurable` entries.

### `FromResource(uri=None, *, ref=None, mime=None, parse=None, required=True, max_bytes=None)`

Dependency-injection marker used inside `typing.Annotated`. Fetches an MCP resource at node entry and parses it into the declared type. The read is *awaited*, so the node must be `async def` and driven by `arun()`; the sync `run()` fails loud. Pass exactly one of `uri=` (a static or RFC-6570-templated URI) or `ref=` (a manifest resource kind to hydrate) — both or neither is a `ConstructError`.

```python
from typing import Annotated
from neograph import node, FromResource

@node(outputs=Assessment)
async def assess(
    doc: Annotated[ContractDoc, FromResource("crm://deals/{deal_id}/contract")],
) -> Assessment: ...
```

- **`uri`** — static or templated resource URI; template vars are interpolated from `config['configurable']` (e.g. sibling `FromInput` values).
- **`ref`** — a manifest `kind`; hydrates the first lifted `ResourceRef` of that kind with layered read/replay/fail-loud expiry.
- **`mime`** — content-type hint for parsing.
- **`parse`** — explicit `(content, mime) -> model` callable; required for `text/*` content unless the parameter is `str`-typed.
- **`required`** — when `False`, a missing fetcher or unmatched ref yields `None` instead of failing loud.
- **`max_bytes`** — fail loud at node entry if the fetched blob exceeds this size, before it reaches a prompt.

Fetches go through the consumer-supplied async `fetch(uri) -> (content, mime)` at `config['configurable']['mcp_resource_fetcher']`; manifest-mode replay uses `config['configurable']['mcp_resource_replayer']` (an async `replay(tool_name, args) -> raw_tool_result`). Full surface: [Resource Hydration](/concepts/resource-hydration/).

### `resource_reader(name, *, uri_template, output_model, description, parse=None, budget=0, idempotent=True)`

Emits a typed, async LangChain `BaseTool` that reads one known resource kind. The tool's argument schema is derived from `uri_template`'s RFC 6570 variables; at call time it reads `config['configurable']['mcp_resource_fetcher']`, fetches the interpolated URI, and parses the blob into `output_model` (via `parse` if given, else `application/json` → `model_validate_json`). The parsed model flows through `ToolInteraction.typed_result`. `idempotent=True` (default) marks the reader replay-safe. Drop it into an agent's `tools=`; async-only, so drive with `arun()`.

### `ResourceRef`, `ProducingCall`, `BlobResult`

Frozen Pydantic models on the resource surface. `ResourceRef` is a lifted `resource_link` (`uri`, `kind`, `server`, `producing_call`, `mime`, `size`, `fetched_at`, `ttl_ms`); `ProducingCall` carries the `(tool_name, args, producer_idempotent)` replay path stamped from the producing `Tool.idempotent` flag; `BlobResult` is the typed escape hatch for opaque content (`uri`, `mime`, `text`, `bytes_b64`, `size`).

## ForwardConstruct

### `class ForwardConstruct(Construct)`

Base class for pipelines with Python control flow. Subclass, declare `Node` attributes, override `forward()`:

```python
class Analysis(ForwardConstruct):
    check   = Node(outputs=CheckResult, prompt='check', model='fast')
    deep    = Node(outputs=Result, prompt='deep', model='reason')
    shallow = Node(outputs=Result, prompt='shallow', model='fast')

    def forward(self, topic):
        checked = self.check(topic)
        if checked.confidence > 0.8:
            return self.shallow(checked)
        else:
            return self.deep(checked)

graph = compile(Analysis())
```

At compile time, NeoGraph traces `forward()` with symbolic proxies. Python `if` compiles to a conditional edge. `for` over a proxy compiles to Each fan-out. `self.loop()` compiles to a graph cycle. `try`/`except` around node calls is allowed but doesn't create a fallback edge in v1 -- see the [ForwardConstruct guide](/forward/branching/) for the full semantics.

Python `for`/`while` loops don't produce graph cycles (same limitation as `torch.jit.trace`). Use `self.loop()` for iterative patterns.

For testing, call `forward()` directly -- it runs real Python with real values.

### `self.loop(body, when, max_iterations=10, on_exhaust="error")`

Explicit loop primitive that compiles to a sub-construct with a `Loop` modifier. Returns a callable; call it with a proxy to wire the loop into the graph.

```python
class Writer(ForwardConstruct):
    draft  = Node(outputs=Draft, prompt='draft', model='fast')
    review = Node(outputs=ReviewResult, prompt='review', model='reason')
    revise = Node(outputs=Draft, prompt='revise', model='reason')

    def forward(self, topic):
        d = self.draft(topic)
        d = self.loop(
            body=[self.review, self.revise],
            when=lambda r: r.score < 0.8,
            max_iterations=5,
        )(d)
        return d
```

- **`body`** -- List of node references (`self.review`, `self.revise`, etc.) that form the loop body. At least one node required.
- **`when`** -- Callable that receives the loop body's latest output. Return `True` to continue looping, `False` to exit.
- **`max_iterations`** -- Iteration cap. Default 10.
- **`on_exhaust`** -- `"error"` (default) raises `ExecutionError`; `"last"` exits with the last result.

Maps 1:1 to `construct | Loop(when=..., max_iterations=...)` in the programmatic API.

## Programmatic IR

### `Node(name, *, mode, outputs, inputs=None, prompt=None, model=None, tools=None, llm_config=None, context=None, skip_when=None, skip_value=None, renderer=None)`

The Node IR class. Used directly for runtime construction, programmatic pipeline generation, and sub-constructs.

```python
decompose = Node("decompose", mode="think", outputs=Claims,
                 prompt="rw/decompose", model="reason")
```

### `Node.scripted(name, fn, inputs=None, outputs=None)`

Convenience constructor for scripted nodes. `fn` is a string key resolved against the `scripted=` dict passed to `compile()`:

```python
def extract_fn(input_data, config): ...

extract = Node.scripted("extract", fn="extract_fn", outputs=RawText)

graph = compile(pipeline, scripted={"extract_fn": extract_fn})
```

### `Construct(name, *, nodes, input=None, output=None, description=None)`

Ordered composition of Nodes. Validates input/output compatibility across the node chain at assembly time.

```python
pipeline = Construct("my-pipeline", nodes=[extract, classify])
```

Sub-constructs need `input=` and `output=` to declare their typed I/O boundary:

```python
enrich = Construct(
    "enrich",
    input=Claims,
    output=ScoredClaims,
    nodes=[lookup, score],
)
```

### Modifiers (pipe syntax)

Apply to a Node or Construct via `|`:

```python
node | Oracle(n=3, merge_fn="combine")
node | Oracle(models=["reason", "fast"], merge_fn="pick_best")
node | Each(over="upstream.field", key="id")
node | Operator(when="condition_name")
node | Loop(when=lambda d: d is None or d.score < 0.8, max_iterations=5)
construct | Loop(when=lambda d: d is None or d.score < 0.8, max_iterations=10)
```

**`Oracle(n=3, models=None, merge_fn=None, merge_prompt=None)`** — N-way ensemble. Exactly one of `merge_fn` / `merge_prompt` required. `models=` assigns model tiers round-robin; infers `n` from `len(models)`. The `merge_prompt` receives upstream context as a dict `{"variants": [...], upstream_key: val, ...}` instead of a bare variant list. Templates use `${variants}` for the variant list and `${upstream.field}` for upstream data.

**`Each(over, key)`** — Fan-out over a dotted-path collection. Results keyed by `getattr(item, key)`.

**`Operator(when)`** — Human-in-the-loop interrupt. `when` is a registered condition name.

**`Loop(when, max_iterations=10, on_exhaust="error", history=False)`** — Cycle modifier. On a Node: self-loop (output feeds back as input). On a Construct: the sub-construct re-runs with its output as the next input. `when` receives the latest output; return `True` to continue. The value may be `None` on the first iteration, so the callable must be None-safe (e.g. `lambda d: d is None or d.score < 0.8`). When `history=True`, each iteration's output is collected in a state list field (`{node_name}_history`), useful for debugging or post-loop analysis.

When `output != input` on a Construct Loop, the sub-construct re-reads original inputs from parent state on each iteration instead of feeding output back. This enables produce+validate patterns where the loop body generates a new result each time and the condition evaluates the output type.

The `|` syntax returns a new Node/Construct with the modifier appended. You can chain: `node | Oracle(...) | Operator(...)`.

### `Node.map(source, *, key) -> Node`

Sugar over `| Each(over=..., key=...)` with IDE-friendly lambda introspection. Returns a new Node with an `Each` modifier appended.

```python
# Lambda form (refactor-safe — Pyright/Pylance catch typos):
verify.map(lambda s: s.make_clusters.groups, key="label")

# String form (escape hatch, equivalent to | Each(...)):
verify.map("make_clusters.groups", key="label")
```

- **`source`** — Either a dotted-path string (`"upstream.field"`) or a lambda that accesses attributes on a state proxy (`lambda s: s.upstream.field`). The lambda must be a pure attribute-access chain; indexing, arithmetic, or underscore-prefixed attributes raise `TypeError`.
- **`key`** — Field on each iterated item used as the dispatch key (same semantics as `Each.key`).

Both forms compile to the same `Each` modifier. The lambda form gives static analysis coverage: renaming an upstream node surfaces as a type-checker error instead of a silent runtime failure.

### `Construct.renderer`

Default renderer propagated to all child nodes that don't have their own renderer set. Dispatch hierarchy: `model.render_for_prompt()` > `node.renderer` > `construct.renderer` > global default > None.

```python
pipeline = Construct(
    "my-pipeline",
    nodes=[extract, classify],
    renderer=JsonRenderer(),
)
```

### `Construct.llm_config`

Default LLM configuration propagated to all child nodes. Per-node `llm_config` merges over the Construct default (node wins on conflicts). Common use: setting `output_strategy="json_mode"` once for a whole pipeline instead of on every node.

```python
pipeline = Construct(
    "my-pipeline",
    nodes=[generate, classify, summarize],
    llm_config={"output_strategy": "json_mode", "temperature": 0.2},
)
```

## Shared

### `compile(...)`

Compiles a `Construct` (or `ForwardConstruct`) into an executable LangGraph `StateGraph`.

```python
compile(
    construct,
    checkpointer=None,
    *,
    llm_factory=None,
    prompt_compiler=None,
    renderer=None,
    cost_callback=None,
    scripted=None,
    conditions=None,
    tool_factories=None,
) -> CompiledGraph
```

- **`construct`** — The `Construct` (or `ForwardConstruct`) to compile.
- **`checkpointer`** — LangGraph checkpointer for persistence/resume. Required when the pipeline uses `Operator` or `interrupt_when=`.
- **`llm_factory`** — Per-tier LLM client factory used by LLM-mode nodes. `llm_factory(tier, ...)` returns a `BaseChatModel`. Required if the pipeline has any `think` / `agent` / `act` node.
- **`prompt_compiler`** — Callable that turns a prompt template + input into a message list for the LLM. `prompt_compiler(template, data, **kwargs)` returns a list of messages. The compiler may accept additional kwargs — `output_model`, `output_schema`, `llm_config`, `context`, `node_name`, `config` — and NeoGraph inspects its signature and passes only the kwargs it declares.
- **`renderer`** — Global renderer used for input rendering when a node has no per-node renderer set.
- **`cost_callback`** — Telemetry hook called after each LLM invocation with token usage.
- **`scripted`** — `dict[str, Callable]` mapping the `fn=` keys used by `Node.scripted(fn=...)` (and `@node(merge_fn=...)`) to their functions.
- **`conditions`** — `dict[str, Callable]` mapping condition names used by `Operator(when=...)` and string-form `interrupt_when=` / `loop_when=` to their predicate functions.
- **`tool_factories`** — `dict[str, Callable]` mapping tool-factory names to `factory(config, tool_config)` callables, for declarative tool lookup.

The LLM kwargs are closed over into a frozen `LlmRuntime` bundle threaded into every factory closure. Two `compile()` calls with different kwargs produce fully isolated graphs. `@node`, `@tool`, and `@merge_fn` register themselves automatically, so `scripted=` / `conditions=` / `tool_factories=` are only needed for the programmatic / runtime-constructed surface.

Retry concerns live in their own layers, not at the `compile()` boundary. See [Retry Semantics](/concepts/retry-semantics/) for the three-layer model (transient -> `llm_factory.with_retry`; output-quality -> `LlmConfig.max_retries`; flaky scripted -> in-function).

### `run(graph, *, input=None, resume=None, config=None, auto_resume=True) -> dict`

Executes a compiled graph. Three modes:

- **`run(graph, input={...})`** — new execution.
- **`run(graph, resume={...}, config=...)`** — continue after an Operator interrupt.
- **`run(graph, config=...)`** — resume from the last checkpoint (crash recovery). Requires a `thread_id` in config and a persistent checkpointer; completed supersteps are skipped. Prefer this over calling the raw graph's `invoke(None, config)` — `run()` runs the checkpoint-schema verify and strips framework internals, which the raw call bypasses.

Returns the final state dict with framework internals stripped.

- **`auto_resume`** — When `True` (default), automatically rewinds to the checkpoint before the earliest changed node when schema fingerprints mismatch. When `False`, raises `CheckpointSchemaError` instead. Based on the Prefect cache-miss model. See [Checkpoint Resume](/concepts/checkpoint-resume/) for details.

### `arun(graph, *, input=None, resume=None, config=None, auto_resume=True) -> dict`

Async twin of `run()`. Same three modes and the same signature; awaits the graph's async I/O internally (`ainvoke` and the async checkpoint helpers). Use it when your `@node` bodies, tools, or `llm_factory` are async, or when you want the pipeline to run cooperatively under an existing event loop.

```python
from neograph import arun

result = await arun(graph, input={"topic": "security"})
```

`arun()` is a drop-in replacement for `run()` in async contexts — it does not require the pipeline itself to be async, but it lets async nodes execute without blocking the loop.

### `stream(graph, *, input=None, resume=None, config=None, auto_resume=True, stream_mode="values")`

Streaming twin of `run()`. Yields chunks as the graph runs instead of returning the final state. Mirrors `run()`'s three modes (new input / resume / crash recovery) and adds `stream_mode` (a str or a list, passed straight to LangGraph — `"values"`, `"updates"`, `"custom"`, `"messages"`, `"debug"`). `values` and `updates` chunks are stripped of framework-internal (`neo_*`) fields; `custom` / `messages` / `debug` payloads pass through untouched. `astream(...)` is the async twin (`async for`).

```python
from neograph import stream, astream

for chunk in stream(graph, input={"topic": "x"}, stream_mode="updates"):
    print(chunk)  # per-node deltas

async for chunk in astream(graph, input={"topic": "x"}, stream_mode="values"):
    print(chunk)  # full state snapshot per step
```

### `emit_progress(event)`

Called from inside a node body (any mode) to push a typed progress/domain event onto the `custom` stream. `event` must be a Pydantic `BaseModel`; it is wrapped in a stable envelope — `{"neograph_event": "progress", "event_type": "<ClassName>", "data": <model_dump>}` — and delivered when the graph is driven by `stream`/`astream` with `"custom"` in `stream_mode`. Under a non-streaming driver it warns once (per process) rather than dropping the event silently.

### Facade verbs

`compile()` returns a `CompiledNeograph` facade over the underlying LangGraph graph. Beyond `run()`/`arun()`/`stream()`/`astream()`, the facade delegates a closed set of methods for inspection, streaming, and state editing — sync: `invoke`, `stream`, `get_state`, `get_state_history`, `get_graph`, `update_state`; async: `ainvoke`, `astream`, `astream_events`, `aget_state`, `aget_state_history`, `aupdate_state`. The raw LangGraph graph is available as `.graph` for anything outside this set.

```python
graph = compile(pipeline, checkpointer=saver)

# Event-level streaming (token/step observability):
async for event in graph.astream_events({"topic": "x"}, config=config, version="v2"):
    ...

# Inspect checkpointed state without re-running:
snapshot = await graph.aget_state(config)

# Edit state before a HITL resume:
graph.update_state(config, {"human_approved": True})
```

### `describe_graph(compiled) -> str`

Returns a Mermaid diagram string for a compiled graph. Paste the output into any Mermaid renderer (GitHub, docs, mermaid.live).

```python
graph = compile(pipeline)
print(describe_graph(graph))
```

### `lint(construct, *, config=None, known_template_vars=None, template_resolver=None) -> list[LintIssue]`

Validates DI bindings in a Construct against a sample config dict. Walks every node (recursing into sub-constructs) and checks that every `FromInput`/`FromConfig` parameter has a matching key in the provided config. Returns a list of `LintIssue` instances; an empty list means all bindings are satisfied. Never raises.

When `config` is `None`, only structural checks are performed: required params are flagged as missing since no config is available.

- **`known_template_vars`** (`set[str] | None`) — Extra variable names the consumer's prompt pipeline provides. These are treated as valid template placeholders in addition to the ones inferred from the construct's state model.
- **`template_resolver`** (`Callable[[str], str | None] | None`) — Maps template-ref names to template text for placeholder validation. When provided, `lint()` resolves each node's `prompt=` template ref through this callable and validates that the template's placeholders are satisfiable.

```python
from neograph import lint

issues = lint(pipeline, config={"topic": "auth", "tracer": my_tracer})
for issue in issues:
    print(f"{issue.node_name}.{issue.param}: {issue.message}")
```

### `LintIssue`

Dataclass representing a single lint problem found by `lint()`:

```python
@dataclass
class LintIssue:
    node_name: str       # which node has the problem
    param: str           # which parameter is unbound
    kind: str            # issue kind (see table below)
    message: str         # human-readable description
    required: bool = False
```

**Issue kinds:**

| Kind | Severity | Meaning |
|------|----------|---------|
| `from_input` | varies | `FromInput` param missing from config |
| `from_config` | varies | `FromConfig` param missing from config |
| `from_input_model` | varies | `FromInput` bundled model field missing from config |
| `from_config_model` | varies | `FromConfig` bundled model field missing from config |
| `loop_condition_unregistered` | ERROR | Loop `when=` string not in condition registry |
| `loop_condition_none_unsafe` | WARN/ERROR | Loop `when=` callable/condition crashes when called with `None` (first iteration value). WARN for callables, ERROR for registered string conditions (which always crash) |
| `resource_hydration_kind_unmatched` | ERROR | `FromResource(ref=<kind>)` param with no upstream `agent`/`act` node able to emit a `resource_link` (manifest guaranteed empty) |

### `parse_condition(expr) -> Callable[[Any], bool]`

Safe expression evaluator for spec-driven conditions. Parses `field op literal` expressions where `op` is one of `< > <= >= == !=` and `literal` is a number, boolean (`true`/`false`), or a quoted string. Dotted field access is supported.

```python
from neograph import parse_condition

check = parse_condition("result.score < 0.8")
check(my_output)  # True if my_output.result.score < 0.8
```

Raises `ValueError` for any expression that does not match the grammar. Used internally by `load_spec` for string-form conditions in pipeline specs; available for custom runtime routing.

### `Tool(name, budget=0, config=None)`

LLM-callable tool with an optional per-call budget. `budget=0` means unlimited.

### `ToolInteraction`

Record of a single tool call during a ReAct loop:

```python
class ToolInteraction(BaseModel, frozen=True):
    tool_name: str
    args: dict[str, Any]
    result: str          # BAML-rendered string (for prompts)
    typed_result: Any    # original Pydantic model (for downstream nodes)
    duration_ms: int
```

`typed_result` preserves the original tool return value. `result` is the BAML-rendered form (via `describe_value`) sent to the LLM in the ReAct loop.

### `@tool(budget=None, config=None)`

Decorator that wraps a function as a `Tool`, auto-registers it, and returns the Tool instance directly:

```python
@tool(budget=5)
def search_codebase(query: str) -> SearchResult:
    """Search the codebase for a query."""
    return SearchResult(file="auth.py", line=42, score=0.95)
```

Tool results can be typed Pydantic models. The framework preserves them in `ToolInteraction.typed_result` and renders them as BAML for the LLM.

### `describe_type(model, *, prefix=...) -> str`

Renders a Pydantic model class as a TypeScript-style (BAML) schema. Used internally for `json_mode` output schemas:

```python
describe_type(SearchResult)
# {
#   file: string
#   line: int
#   score: float  // Relevance score 0-1
# }
```

### `describe_value(instance, *, prefix=...) -> str`

Renders a Pydantic model instance in the same BAML notation with actual values:

```python
describe_value(SearchResult(file="auth.py", line=42, score=0.95))
# {
#   file: "auth.py"
#   line: 42
#   score: 0.95  // Relevance score 0-1
# }
```

Used for tool result rendering in ToolMessage content.

### `render_prompt(node, input_data, *, config=None) -> str`

Returns the exact prompt that would be sent to the LLM, without calling the LLM. For debugging and testing prompt templates.

### `compile_prompt(template, input_data, *, output_model=None, output_schema=None, di_inputs=None, prompt_compiler=None, renderer=None, node_name="", template_text=None, loader=None) -> list[dict]`

Compiles a prompt to a message list **standalone** — no compiled graph, no run — producing byte-identical messages to what a compiled `think` node sends for the same inputs (same renderer dispatch, same schema injection, same `di_inputs` layering). It routes through the exact internal seam the graph uses, so an eval harness tests the REAL prompt instead of a re-implementation. See [Evaluating your prompts](/concepts/evaluating-prompts/).

- **`template`** — the template name a file-ref `prompt_compiler` resolves (e.g. `"rw/classify"`), or an inline template (`"Summarize ${topic}"`, no compiler needed).
- **`input_data`** — the input the node consumes: a Pydantic model, a dict of upstream-name → value (fan-in), or `None` for an all-DI leaf node.
- **`output_model`** / **`output_schema`** — inject the output schema (a `describe_type` string) into the compiler's schema var. Pass `output_model` (schema derived) or a precomputed `output_schema`.
- **`di_inputs`** — resolved `FromInput`/`FromConfig` values keyed by parameter name; layered as the base namespace (upstream outputs shadow on a name collision), exactly as the graph does.
- **`prompt_compiler`** — the file-ref compiler. Pass the SAME one production uses for byte-identity. Not needed for an inline template.
- **`renderer`** — the input renderer (`None` → BAML `describe_value` default).
- **`node_name`** — threaded to a `render_messages` override for per-node role shaping.
- **`template_text`** / **`loader`** — the template-source override for eval variants: wrap a raw variant string (`template_text`) or a `name → text` loader in a `DefaultPromptCompiler` so a harness can parameterize the filename-versioned variant convention. Mutually exclusive with `prompt_compiler`.

### `RenderedInput`

Dataclass bundling rendered prompt input for a node. Provides both raw Pydantic models and their rendered string forms, letting the factory choose the right representation for inline vs template-ref prompts.

```python
@dataclass
class RenderedInput:
    raw: dict[str, Any] | Any
    rendered: dict[str, Any] | Any
    flattened: dict[str, Any]
    available_keys_inline: set[str]
    available_keys_template: set[str]
```

- **`raw`** — Raw Pydantic models (or dicts) for inline `${var}` dotted access in prompts.
- **`rendered`** — BAML-rendered strings for template-ref prompts (each model passed through `describe_value`).
- **`flattened`** — Extra fields surfaced by `render_for_prompt()` BaseModel returns. Merged into the template namespace.
- **`available_keys_inline`** — Valid variable names for inline prompts (`${key}` or `${key.field}`).
- **`available_keys_template`** — Valid variable names for template-ref prompts.
- **`for_template_ref`** (property) — Merges `rendered` and `flattened` dicts into a single namespace for template-ref rendering.

### `build_rendered_input(input_data, renderer=None) -> RenderedInput`

Constructs a `RenderedInput` from raw input data. Applies the optional renderer (or BAML default) to produce rendered string forms of each input model. Used internally by the factory; available publicly for prompt debugging and custom integrations.

### `ExcludeFromOutput`

Marker for `typing.Annotated` on Pydantic model fields. Fields annotated with `ExcludeFromOutput` are visible in input rendering (the LLM sees them as context) but excluded from the `describe_type()` output schema, so the LLM will not attempt to produce them. Fields with this marker must have a default value.

```python
from typing import Annotated
from neograph import ExcludeFromOutput

class AnalysisResult(BaseModel):
    summary: str
    confidence: float
    source_doc: Annotated[str, ExcludeFromOutput] = ""  # visible as input, hidden from output schema
```

### Error Hierarchy

Parentage follows one rule, by lifecycle phase: a failure raised **while the graph is executing** subclasses `ExecutionError`; an **assembly/validation** failure is a `ConstructError`; a **compile()** failure is a `CompileError`; a **bad or missing configuration** (including a resume-time precondition mismatch) is a `ConfigurationError`. This means a single `except ExecutionError` around `run()` catches every runtime failure — including resource-expiry and non-idempotent-replay.

```
NeographError (base)
  ├── ConstructError (ValueError)  — assembly-time validation failures
  ├── CompileError                 — compilation failures
  ├── ConfigurationError           — missing LLM config, unregistered tools
  │     └── CheckpointSchemaError  — checkpoint state schema mismatch on resume
  └── ExecutionError               — runtime failures during a run
        ├── PromptVarMissing          — strict prompt placeholder had no value
        ├── StateMissingError         — required state read missed
        ├── NodeOutputError           — a node ran and produced None vs its declared output
        ├── NonIdempotentReplayError  — refused to replay a non-idempotent producing tool
        └── ResourceExpiredError      — a manifest ResourceRef could not be hydrated
```

**`CheckpointSchemaError`** — raised when resuming from a checkpoint whose state schema fingerprint differs from the current graph. Has an `invalidated_nodes: set[str]` attribute listing which state fields changed. Only raised when `auto_resume=False`; when `auto_resume=True` (the default), `run()` automatically rewinds instead. It subclasses `ConfigurationError`, not `ExecutionError`: it fires at resume, before any node re-executes, as a precondition check on the persisted checkpoint against the current graph — a configuration mismatch you resolve with a new `thread_id` or a schema migration, not a retry of node logic.

All errors are importable: `from neograph import ConstructError, CompileError, ConfigurationError, CheckpointSchemaError, ExecutionError, PromptVarMissing, NonIdempotentReplayError, ResourceExpiredError`.

### Runtime wiring (compile() dict kwargs)

The programmatic / runtime-constructed surface references scripted functions, conditions, and tool factories by string name. Supply the implementations as dicts to `compile()` — there are no global registration functions:

- **`scripted={"name": fn}`** — Resolves `Node.scripted(fn="name")` (and `@node(merge_fn="name")`).
- **`conditions={"name": fn}`** — Resolves `Operator(when="name")` and string-form `interrupt_when="name"` / `loop_when="name"`.
- **`tool_factories={"name": factory}`** — Resolves declarative tool lookup; `factory(config, tool_config)` returns a tool.

```python
graph = compile(
    pipeline,
    scripted={"extract_fn": extract_fn, "merge_claims": merge_claims},
    conditions={"needs_review": needs_review},
    tool_factories={"search_code": lambda config, tool_config: SearchTool()},
)
```

These are only needed for the programmatic API. `@node`, `@tool`, and `@merge_fn` handle wiring automatically.

## Spec Loader

### `load_spec(spec, project=None)`

Load a YAML/JSON pipeline spec and return a compilable `Construct`.

```python
from neograph import load_spec, compile, run

construct = load_spec("pipeline.yaml")
graph = compile(construct)
result = run(graph, input={"node_id": "demo"})
```

**Parameters:**

- **`spec`** (`str | dict`) — Pipeline spec as a YAML/JSON string, a file path, or a pre-parsed dict.
- **`project`** (`str | dict | None`) — Project surface (types/tools/models) as a YAML/JSON string, file path, or pre-parsed dict. Optional — types can also be pre-registered via `register_type`.

**Returns:** A `Construct` ready for `compile()`.

The spec is validated against the bundled JSON Schema (`neograph/schemas/neograph-pipeline.schema.json`) when `jsonschema` is installed. After parsing, the standard Construct validator runs — same checks as `@node` and programmatic pipelines.

See [Pipeline Spec Format](/concepts/spec-format/) for the full spec schema and examples.

### `register_type(name, cls)`

Register a Pydantic model under a string name for spec-based lookup.

```python
from neograph import register_type
from myapp.schemas import Draft

register_type("Draft", Draft)
```

Types referenced in specs (e.g., `outputs: Draft`) are resolved from this registry. Types can also be auto-generated from a project surface passed to `load_spec(project=...)`.

### `lookup_type(name)`

Return the model registered under the given name. Raises `ConfigurationError` if not found.
